۱ نمای کلی محصول

هدف: ساخت یک CRM سازمانی فارسی، چندکاربره، ماژول‌محور و قابل سفارشی‌سازی — معادل عملکردی BaMaCRM (بر پایه Vtiger).

این محصول یک «CRM ساده» نیست. یک پلتفرم Enterprise با موتور متادیتا، RBAC سه‌لایه، موتور اتوماسیون، و ۳۰+ ماژول است.

اصول طراحی (حتماً رعایت شود)

🔧 Metadata-First

هیچ ماژولی hard-code نشود. فیلدهای سفارشی، layout و دسترسی از متادیتا خوانده شود.

🔐 Row-Level Security

فیلتر دسترسی در لایه Query، نه UI. بر اساس مالک، ارجاع، نقش و Sharing Rules.

🏢 Multi-Tenant Ready

از روز اول tenant_id در همه جداول. هر سازمان داده جدا.

🔌 API-First

REST API v1 برای یکپارچگی مشتریان + Webhook inbound/outbound + Swagger/OpenAPI + API Key با Scope.

Metadata Engine (قلب سیستم)

هر ماژول از این موتور استفاده می‌کند:

  • modules — تعریف ماژول (name, label, icon, table_name)
  • fields — فیلدها (type, required, hidden, default, unique)
  • picklist_values — مقادیر لیست کشویی
  • layouts — چیدمان فرم (sections, field order)
  • relations — ارتباط بین ماژول‌ها (lookup, master-detail)

فیلدهای سفارشی در ستون custom_data JSONB هر جدول ذخیره می‌شوند (حداکثر ۱۵ فیلد).

۵ سیستم دسترسی (RBAC سه‌لایه)

لایه ۱ — نقش‌ها (Roles)

  • ساختار درختی: parent_role_id (مدیرعامل ← مدیر فروش ← کارشناس)
  • تنظیم «قابلیت ارجاع به»: همه کاربران / هم‌سطح و پایین‌تر / فقط پایین‌تر
  • هنگام حذف نقش: انتقال داده به نقش دیگر

لایه ۲ — پروفایل‌ها (Profiles)

دسترسی per-module: View | Create | Edit | Delete

دسترسی per-field: Hidden | ReadOnly | Editable

امکانات اضافی: Import | Export | Merge | Duplicate Detection

لایه ۳ — Sharing Rules

حالتتوضیح
Privateفقط مالک + زیرمجموعه نقش + ارجاع‌شده
Public Read Onlyهمه می‌بینند، فقط مالک ویرایش
Public Read/Writeهمه می‌بینند و ویرایش، فقط مالک حذف
Public Fullهمه دسترسی کامل
Sharing RulesPrivate / Public-RO / Public-RW / Public-Full + Custom
Sharing Rules — Calendarهمان ۴ حالت، جداگانه per-calendar module

الگوریتم فیلتر Query

WHERE tenant_id = :tenant
  AND deleted_at IS NULL
  AND (
    is_admin = true
    OR created_by = :userId
    OR assigned_to = :userId
    OR created_by IN (subordinate_user_ids)
    OR id IN (sharing_rule_granted_ids)
  )
